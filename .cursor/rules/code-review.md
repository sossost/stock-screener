# 코드 리뷰 자동화 규칙

## 작업 완료 전 필수 검증

코드 작성/수정 후 반드시 다음을 확인:

### 1. 컴포넌트 분리 검증

- 인라인 렌더링 로직이 3회 이상 반복되는가?
- 공용 컴포넌트(`Button`, `PageHeader` 등)를 사용하는가?
- 유틸 함수로 분리 가능한 로직이 있는가?

### 2. 타입 안전성 검증

- `any` 타입 사용 여부
- Non-null assertion(`!`) 남용 여부
- 타입 중복 정의 여부

### 3. 에러 핸들링 검증

- 낙관적 업데이트 시 롤백 로직 존재 여부
- 입력 검증(음수, NaN, 범위) 체크
- 에러 전파 또는 사용자 피드백

### 4. UI/UX 안전성 검증

- 절대 위치 요소의 경계 체크
- 날짜/숫자 파싱 에러 처리
- 빈 상태 처리

### 5. 폴더 구조 검증

- 관련 컴포넌트의 논리적 그룹화
- Import 경로 일관성

## 자동 검증 명령어

```bash
# 1. 린트 검사
yarn lint

# 2. 타입 체크 + 빌드
yarn build

# 3. 테스트
yarn test
```
